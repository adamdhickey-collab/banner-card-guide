Character Count Limits — Both Card Types

Use these limits as Sitecore field validators. "Max" is the hard limit; "Recommended" is the editorial sweet spot.

Type A & Type B — Shared Character Count Limits
Field Recommended Hard Max Notes
Title 15–35 chars 45 Titles over ~30 chars will wrap to 2 lines
Description 80–150 chars 200 Type A cards tend to read better with shorter copy
Primary CTA Button 18–28 chars 35 Applies to both card types
Secondary Link 18–28 chars 35 Optional on both types. Include a trailing › chevron.

Title at max (45 chars):
"Send and Receive Money Safely with Zelle® App"

Description at max (200 chars):
"With Bill Pay, control when and how you pay your bills — plus, pay individuals, companies, send a personalized gift check or donate to your favorite charity. Set up recurring payments with ease and peace of mind."

Primary CTA at max (35 chars):
"View QuickStep Business Loan Details"

Secondary Link at max (35 chars):
"Watch the electronic statement & notices demo"

Secondary link is optional on both card types. When omitted, an empty reserved zone maintains button alignment across all cards in the row.

Developer Implementation Notes — Sitecore SXA

How to achieve equal-height rows, pinned CTAs, and responsive column behaviour without fixed heights.

The Core Flexbox Pattern (applies to both types)

This is the key technique. Do not set a fixed height on the card — let flexbox do the work:

  • Card wrapper: display: flex; flex-direction: column;
  • Description element: flex: 1; — absorbs all remaining vertical space
  • Actions wrapper: margin-top: auto; — forces CTAs to the card bottom
  • The grid itself provides equal-height rows via the default align-items: stretch
⚠ Never set height, min-height, or max-height on the card element itself. Fixed heights cause unnecessary whitespace on mobile when a single-column layout makes content much shorter.

Responsive Grid

  • Desktop (≥ 901px): grid-template-columns: repeat(3, 1fr)
  • Tablet (561–900px): grid-template-columns: repeat(2, 1fr)
  • Mobile (≤ 560px): grid-template-columns: 1fr
  • SXA Bootstrap equivalent: col-lg-4 col-md-6 col-12 inside a .row with d-flex flex-wrap
  • If using Bootstrap rows, add align-items: stretch to the .row so cards in the same row grow to equal height

Type A — Rectangular Image

  • Image dimensions: 600 × 338px minimum (16:9 ratio), 1200 × 675px ideal for retina
  • Use aspect-ratio: 16 / 9; object-fit: cover; on the image — this enforces the ratio regardless of the uploaded image's natural dimensions
  • Image sits outside the padded body — flush to top, left, and right edges of the card
  • Card body padding: 24px all sides
  • Sitecore rendering parameter for image URL: ?w=600&h=338&as=1&mode=crop

Type B — Circular Image

  • Image dimensions: 240 × 240px minimum (square source crop)
  • Display size: width: 118px; height: 118px; border-radius: 50%;
  • Always use object-fit: cover so off-square uploads still render as a clean circle
  • Sitecore rendering parameter: ?w=240&h=240&as=1&mode=crop
  • All text and CTAs are text-align: center; the .card-b__actions wrapper uses display: flex; flex-direction: column; align-items: center;

Sitecore SXA Component Mapping

  • Use the built-in Promo rendering as the base for both card types — it provides image, title, text, and a primary link field out of the box
  • For Type B's secondary link: add a custom "Secondary Link" field to the Promo data template (/sitecore/templates/Feature/Experience Accelerator/Page Content/Promo/...) and render it conditionally in the Rendering Variant
  • Create a Rendering Variant for each type (e.g., Type A - Rectangular and Type B - Circular) to swap the layout without a separate component
  • Use a Page List or Carousel rendering with a Promo variant to auto-populate from a datasource folder if cards are content-managed centrally

Character Count Validation in Sitecore

  • Apply Sitecore's built-in "Maximum length" field validator to each content field in the data template
  • Title field: set max length to 45 (covers both types; stricter editorial guidance handles the 40-char Type A limit)
  • Description field: set max length to 200
  • Link text fields: set max length to 40
  • Consider adding a Warning threshold (e.g., 80% of max) so authors see a caution before hitting the hard limit

Typography

  • Font: Nunito Sans is the closest freely available match to Museo Sans — load via Google Fonts or self-host for GDPR compliance
  • Required weights: 400 (body), 600 (semi-bold), 700 (title), 800 (CTA, badge)
  • Fallback stack: 'Nunito Sans', 'Segoe UI', Arial, sans-serif
  • If licensing Museo Sans directly, load via @font-face in your SXA theme's fonts.css

Design Tokens / CSS Variables

  • Define all colours and spacing as CSS custom properties on :root in your SXA theme's base.css
  • CTA button background: --color-cta-bg: #fec938
  • CTA hover state: --color-cta-bg-hover: #e5b425
  • CTA border: none
  • Secondary link: --color-link: #0160af
  • Card background: --color-card-bg: #ffffff
  • Card border: 1px solid #d8d8d8

Accessibility

  • Every <img> must have a meaningful alt attribute — author-entered in Sitecore's Image field
  • CTA button text must be unique per card (screen readers announce all buttons; "Learn more" repeated 6× is not accessible)
  • Secondary link chevron (›) should be wrapped in <span aria-hidden="true"> so it is not read aloud
  • Colour contrast: #1a1a1a on #F2A900 meets WCAG AA (4.7:1 ratio)
